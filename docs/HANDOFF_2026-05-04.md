## Bugs Fixed This Session

### 1. `<think>` blocks consuming all `max_tokens` → empty responses

**Root cause:** `_stream_llamacpp_tokens` used `max_tokens: 2048`. Hermes 4.3 36B's hybrid
thinking mode can generate 1000–2000 tokens of `<think>` content, using up the full budget before
generating any actual response. Client received an empty stream.

**Fix:** [src/guppy/inference/streaming_backends.py:305](src/guppy/inference/streaming_backends.py#L305) — `max_tokens` raised from `2048` → `4096`.

---

### 2. `first_response` fallback check failing for think-only responses

**Root cause:** In both companion and workspace `_generate()` in
[src/guppy/api/routes_realtime.py](src/guppy/api/routes_realtime.py), `first_response`
accumulated RAW tokens (including `<think>` content). The fallback check
`if not first_response.strip()` never fired because think content is non-empty.
Meanwhile `_think_filter` in `_drain()` stripped ALL visible tokens → client got nothing.

**Fix (companion, line ~1074):** Strip think blocks from `first_response` BEFORE the fallback check and tool detection:
```python
first_response_clean = _strip_think_from_text(first_response)
if not first_response_clean.strip():
    first_response_clean = "I'm here — could you rephrase that?"
    yield f"data: {json.dumps({'token': first_response_clean})}\n\n"
tool_blocks = _TOOL_CALL_RE.findall(_normalize_tool_calls(first_response_clean))
```
Same fix applied to workspace pass-1.

---

### 3. `follow_up_history` contaminated with raw `<think>` content

**Root cause:** When a tool call was detected and pass-2 was initiated, the
`follow_up_history` injected the raw `first_response` (containing full `<think>` block) as the
assistant's turn. This polluted the pass-2 context with hundreds of tokens of model reasoning
noise.

**Fix:** Both companion and workspace now pass `first_response_clean` (stripped) into
`follow_up_history`.

---

### 4. `/no_think` suppression for companion surface

**Root cause:** Hermes 4.3 36B (ByteDance Seed OSS architecture) supports hybrid thinking mode.
The system-prompt directive "Do NOT use `<think>` blocks" was being ignored by the model.

**Fix:** [src/guppy/inference/router_surface.py](src/guppy/inference/router_surface.py) — Added
`_sp_suppress_think()` helper that appends ` /no_think` to the last user message before sending
to hermes4 on companion surface. Also added `thinking_budget=0` parameter to
`_stream_llamacpp_tokens` which passes `budget_tokens: 0` in the API payload (llama.cpp Seed
model support for thinking budget).

---

### 5. Memory upsert-by-key (stale preference accumulation)

**Root cause:** `_remember_sqlite` in [src/guppy/memory/semantic.py](src/guppy/memory/semantic.py)
only deduplicated by VALUE (same value within 24h). Storing a preference with the same KEY but
updated VALUE would create a second record, causing both old and new values to exist in DB.

**Fix:** Added DELETE before INSERT for the same `memory_key`. Now: if you store
`"preferences.concise"` → old entry is removed, new one inserted. Value-dedup still applies (skip
if exact same value exists within 24h).

---

### 6. Stale test references to `llamacpp-rocinante`

**Root cause:** [tests/integration/test_usability_blockers.py](tests/integration/test_usability_blockers.py) referenced `llamacpp-rocinante` context window (16K) and budget threshold (85%). Rocinante was removed in the single-model consolidation.

**Fix:** Updated to `llamacpp-hermes4` (128K), budget threshold 60% (matching the current
`stream_unified_inference` guard), and recalculated the "very long session" threshold accordingly.
