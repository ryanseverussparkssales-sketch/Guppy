# Guppy Vision Board

## North Star
Build a hybrid AI butler and coding partner that can execute work safely, preserve context, and increase output quality and revenue while reducing manual overhead.

## Who This Is For
- Ryan in active work mode who needs delegation without losing control
- A freelancer/operator who needs communication, execution, and delivery support
- A builder who wants research-to-app-to-report flows in one assistant

## Core Promise
- Handle practical PC and workflow tasks while Ryan stays focused
- Deliver concise, useful responses with minimal backend noise
- Remember important context reliably across sessions
- Speak only useful persona content and stop voice when user re-engages

## Capability Pillars

### 1. Executive Butler Ops
- PC task execution and lightweight desktop control
- Email drafting, cleanup, and account-aware actions
- Reminder and schedule support
- Priority and commitment tracking

### 2. Builder and Coding Partner
- Repo-aware coding and debugging support
- App and script generation from requirements
- Fast conversion from idea to implementation checklist
- Report and asset generation for client delivery

### 3. Revenue and CRM Motion
- Lead and opportunity tracking
- Follow-up and next-action workflows
- Sales report generation and outreach assistance
- Progress visibility toward close and value targets

### 4. Research and Library Intelligence
- Web research and source synthesis
- Targeted scraping workflows
- Media and file organization support
- Context retrieval from past sessions and notes

### 5. Adaptive Memory and Behavior
- Persistent memory with deterministic recall
- Preference-aware response style
- Session continuity and summarized history
- Safe learning via explicit user-confirmed patterns

## Market Comparison Snapshot

### Typical Coding Copilots
Strength: code generation and debugging
Gap vs vision: weak personal ops, scheduling, and business workflow ownership

### Typical Productivity Assistants
Strength: calendar/email/task convenience
Gap vs vision: weak coding depth, desktop tooling, and local control

### Typical CRM Platforms
Strength: pipeline and account management
Gap vs vision: weak conversational execution and personal assistant behavior

- Output quality and UX polish: strong progress

## Current Operating Mode (April 11, 2026)
- Revenue expansion is paused temporarily.
- Active execution priority is launch stability, voice reliability, and clean handoff continuity.
- Claude budget constraints are treated as an operational condition, not a product-level blocker.
- Voice stack should continue to function in fallback mode when optional packages are unavailable.

## 12-24 Month Tech Direction
- Hybrid local plus cloud model routing
- Multimodal desktop grounding with strict safety boundaries
- Policy-based autonomy levels and explicit confirmations
- Rich memory graph for people, projects, commitments, and outcomes
- Operational telemetry for trust, reliability, and optimization loops

## Success Metrics
- Time saved per day and per week
- Revenue actions completed and follow-up compliance
- Task completion rate and overdue reduction
- Recall precision and context relevance
- User interruption rate and handoff smoothness
- Assistant action acceptance vs rejection ratio

## Guardrails
- Confirm destructive actions
- Keep user-visible output concise by default
- Separate persona response from backend/tool details
- Provide transparent thinking summaries without chain-of-thought leakage
- Allow immediate user override at any point
