WHAT WAS IMPLEMENTED
================================================================================

A smart inference routing system that automatically selects the best model:

PRIMARY:    Local guppy model (Ollama)
            - Free ($0/month)
            - Fast (100-150 tokens/sec)
            - 30-second timeout

FALLBACK 1: Claude Haiku (cloud)
            - Low cost (~$0.80 per million tokens)
            - Fast enough (1-3 seconds latency)
            - Reliable cloud backup

FALLBACK 2: Claude Sonnet (cloud)
            - Medium cost (~$3 per million tokens)
            - Highest reasoning capability
            - Last resort for complex queries

HOW IT WORKS
================================================================================

When a user sends a message via the /chat endpoint:

1. Request arrives at guppy_api.py
2. guppy_api.py calls _call_unified_inference()
3. _call_unified_inference() uses InferenceRouter
4. Router.query() follows priority chain:
   
   Step 1: Try LOCAL (guppy model via Ollama)
           - Sends request to http://127.0.0.1:11434/api/chat
           - Waits up to 30 seconds
           - If success → RESPOND TO USER
           - If timeout/error → Go to Step 2

   Step 2: Try HAIKU (Claude Haiku via Anthropic API)
           - Sends request to Claude API
           - Usually responds in 1-3 seconds
           - If success → RESPOND TO USER
           - If error → Go to Step 3

   Step 3: Try SONNET (Claude Sonnet via Anthropic API)
           - Sends request to Claude API
           - Highest reasoning capability
           - If success → RESPOND TO USER
           - If error → RAISE ERROR (shouldn't happen)

5. Logs which model was used
6. Returns response to user

Result: User always gets an answer, using the cheapest/fastest available model.

LOGGING & MONITORING
================================================================================

The router logs every inference request. Watch for:

[LOCAL] ✓ Success
  → Guppy model was used (optimal, free)

[LOCAL] Timeout (30s). Falling back.
  → Local model didn't respond in time, trying Haiku

[HAIKU] ✓ Success. Tokens: 245
  → Haiku was used (fallback worked, low cost)

[HAIKU] Error: [reason]. Falling back.
  → Haiku failed, trying Sonnet (rare)

[SONNET] ✓ Success. Tokens: 512
  → Sonnet was used (last resort, high cost/quality)

[ERROR] Critical error
  → All models failed (shouldn't happen)

HEALTHY PATTERN:
  Mostly [LOCAL] messages = System working as designed (optimal)
  Occasional [HAIKU] messages = Normal fallback behavior
  Rare [SONNET] messages = Conservative fallback working

CONFIGURATION
================================================================================

All configuration is in inference_router.py:

TIMEOUT (line 23):
  OLLAMA_TIMEOUT = 30
  → How long to wait for local model before fallback
  → Increase if local model is slow
  → Decrease for faster cloud fallback

MODEL NAMES (lines 24-27):
  LOCAL_MODEL = "guppy"
  HAIKU_MODEL = "claude-3-5-haiku-20241022"
  SONNET_MODEL = "claude-3-5-sonnet-20241022"
  → Can customize if using different models

FALLBACK PREFERENCE (query() method):
  Default: Try Haiku before Sonnet (cost optimization)
  Can change via prefer_fallback parameter

OLLAMA API ENDPOINT (line 18):
  OLLAMA_API = "http://127.0.0.1:11434/api/chat"
  → Local Ollama server endpoint
  → Change if Ollama is on different machine/port
