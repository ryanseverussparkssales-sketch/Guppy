# Instructions Folder

This folder is the canonical operator and contributor instruction set.

If guidance here conflicts with older files under `docs/` or root markdown,
follow this folder first.

## Files

- `instructions/OPERATIONS.md`
  - Day-to-day runbook (startup, health checks, recovery, overnight)
- `instructions/DEVELOPMENT.md`
  - Developer workflow, quality gates, and PR expectations

## Scope Contract

- `instructions/` owns executable process instructions.
- `documentation/` owns architecture, security, and truth-audit references.
- Historical notes can remain in root/docs for archive value, but are non-canonical.
